# 프롬프트 템플릿 모음

## 1. 웹 리서치 프롬프트

```markdown
주제: [주제]

배경:
사용자가 [사용자]로서 [목적]을 위해 이 정보를 찾고 있어요.

리서치 질문:
1. [구체 질문 1]
2. [구체 질문 2]
3. [구체 질문 3]

출력 형식:
- 출처와 함께 정리해줘
- 숫자로는 구체적으로 표시
- 불확실한 부분은 "확인 필요"로 표시

제약사항:
- 2026년 이후 자료 우선
- 실제 사용 가능한 방법 위주
```

## 2. 코드 작성 프롬프트

```markdown
작업: [작업 설명]

요구사항:
- [요구 1]
- [요구 2]
- [요구 3]

출력 형식:
1. 코드 블록 (```lang)
2. 간단한 설명
3. 사용 방법 (선택)
4. 주의사항

테스트 방법:
1. [테스트 방법]
2. [테스트 방법]

버전 관리:
- 초기 버전: v1.0
- 마지막 수정일: [날짜]
```

## 3. 요약 프롬프트

```markdown

1. 3문장 이내로 요약

## 핵심 요약

3. [액션 3]

### 참고 링크
- [링크 1]
- [링크 2]
```

## 5. 에러 해결 프롬프트

```markdown
에러 발생:
[에러 내용]

컨텍스트:
- 작업 중이던 것: [작업]
- 에러 발생 단계: [단계]
- 재현 방법: [방법]

요청:
1. 원인 분석
2. 해결 방법 (순서대로)
3. 예방 방법
4. 대안 (있으면)

출력 형식:
## 원인
[에러 원인]

## 해결 방법
1. [단계 1]
2. [단계 2]
3. [단계 3]

## 예방 방법
[예방 방법]

## 대안
[대안]
```

3. [단계 3]

## 8. 팩트체크 프롬프트

```markdown
팩트체크 요청

## 대상 내용
[팩트체크할 내용]

## 검사 항목
- [ ] 주장에 근거가 있는가?
- [ ] 출처가 명시되었는가?
- [ ] 숫자가 정확한가?
- [ ] 논리에 약점이 있는가?

## 출력 형식
## 팩트체크 결과

### 근거 확인
- [주장] → [확인 결과]
- [주장] → [확인 결과]

### 출처 확인
- [주장] → [출처] / [출처 없음]
- [주장] → [출처] / [출처 없음]

### 숫자 확인
- [숫자] → [확인 결과]

### 논리 검토
- [논리] → [검토 결과]

### 개선 제안
[개선 제안]
```

## 사용 팁

### 프롬프트 작성 원칙
1. **명확하고 구체적**: 모호한 지시 피하기
2. **예시 포함**: 원하는 형식 보여주기
3. **제약사항 명시**: 무엇을 하지 말아야 하는지
4. **출력 형식 정의**: 결과물의 구조 정하기
5. **컨텍스트 제공**: 배경과 목적 명시하기

### 템플릿 활용 방법
1. 템플릿 복사
2. `[ ]` 부분 채우기
3. 필요 시 수정
4. 사용
