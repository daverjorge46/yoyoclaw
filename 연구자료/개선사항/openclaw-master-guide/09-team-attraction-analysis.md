# 부록 9: 팀 매력 분석 + 작업 이슈 해결법

## 현재 세나팀 분석

### 강점
| 항목 | 현재 상태 | 평가 |
|------|----------|------|
| 역할 분담 | 명확 (리서처/빌더/비평가/종합) | ✅ 우수 |
| 캐릭터 깊이 | 나이/성격/관계 설정 있음 | ✅ 우수 |
| 토론 구조 | 라운드제 (최대 3R) | ✅ 체계적 |
| 사용자 보고 | 구조화된 포맷 | ✅ 명확 |
| 상호작용 | sessions_send 기반 A2A | ⚠️ 개선 필요 |

### 개선 필요 영역

#### 1. 실시간 상호작용 부족
**현상**: 팀원 간 직접 대화가 sessions_send로만 이루어짐. 그룹 채팅에서의 자연스러운 대화 흐름이 부족.
**개선안**:
- 텔레그램 그룹에서 멘션 기반 직접 소통
- 팀원이 다른 팀원의 결과에 자발적으로 반응
- "미루가 올린 리서치에 유리가 바로 팩트체크" 같은 자연스러운 흐름

#### 2. 비동기 작업 관리
**현상**: 딥워크 모드에서 팀원 결과물이 언제 완성되는지 추적 어려움
**개선안**:
- 각 팀원의 작업 상태를 MEMORY.md에 기록
- cron 기반 진행 상황 체크
- 팀원이 작업 완료 시 세나에게 자동 알림

#### 3. 결과물 품질 일관성
**현상**: 팀원별 출력 형식/깊이가 다름
**개선안**:
- 팀 공통 출력 템플릿 정의
- 유리(비평가)의 품질 검토를 필수 단계로
- 체크리스트 기반 품질 검증

#### 4. 컨텍스트 공유
**현상**: 팀원 간 작업 맥락이 충분히 공유되지 않음
**개선안**:
- 공유 워크스페이스 파일 활용
- 작업 결과를 파일로 저장하고 서로 참조
- 팀 브리핑 노트 자동 생성

## 작업 중 발생한 이슈 분석 & 해결법

### 이슈 1: 팀원 결과물 파일 접근 불가
**상황**: 하나가 "파일 완료"라고 보고했지만 실제 파일이 데스크탑에 없음
**원인**: 각 에이전트가 독립 세션에서 실행되어 파일 생성이 다른 경로에 될 수 있음
**해결법**:
1. 파일 저장 경로를 명시적으로 지정 (절대 경로)
2. 세나가 파일 존재 확인 후 다음 단계 진행
3. 팀 공유 워크스페이스 경로 통일:
```json
{
  "routing": {
    "agents": {
      "sena": { "workspace": "~/.openclaw/agents/sena/workspace" },
      "miru": { "workspace": "~/.openclaw/agents/miru/workspace" },
      "hana": { "workspace": "~/.openclaw/agents/hana/workspace" },
      "yuri": { "workspace": "~/.openclaw/agents/yuri/workspace" }
    }
  }
}
```
4. 공유 출력 폴더 설정: 모든 팀원이 동일 경로에 결과물 저장

### 이슈 2: 라운드 진행 지연
**상황**: R1에서 미루만 결과를 보내고 하나/유리 응답이 늦음
**원인**: sessions_send 기반 통신에서 응답 대기 시간이 길 수 있음
**해결법**:
1. 각 팀원에게 타임박스 설정 (예: R1은 15분 이내)
2. 병렬 작업 시 결과 도착 순서에 따라 유연하게 진행
3. "먼저 온 결과부터 교차 검토 시작" 전략
4. 세나가 주기적으로 상태 체크

### 이슈 3: 토론 수렴 실패
**상황**: R2에서 팀원들의 의견이 평행선
**원인**: 판단 기준이 불명확하거나, 데이터 부족
**해결법**:
1. 세나가 명확한 판단 기준 제시
2. 추가 데이터 요청 (미루에게)
3. 세나가 최종 판단 후 "합의 선언"
4. 최대 3라운드 규칙 엄수

### 이슈 4: 출력물 중복/누락
**상황**: 여러 팀원이 같은 내용을 다루거나, 중요 부분이 빠짐
**원인**: 작업 분배가 불명확하거나, 경계가 모호
**해결법**:
1. 킥오프에서 각 팀원의 담당 범위를 구체적으로 명시
2. 체크리스트로 필수 항목 관리
3. 세나가 R1 후 누락/중복 확인 후 재배분

### 이슈 5: 딥워크 모드 시간 관리
**상황**: 5시간 딥워크인데 라운드별 시간 배분이 어려움
**원인**: 라운드 수와 각 라운드 소요 시간 예측 어려움
**해결법**:
1. 시간 배분 템플릿:
   - R1 (리서치): 전체 시간의 40% (2시간)
   - R2 (교차 검토): 25% (1시간 15분)
   - R3 (보완): 20% (1시간)
   - R4 (종합 정리): 15% (45분)
2. 각 라운드 시작 시 타이머 설정
3. 중간 보고로 진행 상황 가시화
