# 섹션 2: 활용 팁 & 베스트 프랙티스

## 2.1 Top 10 파워유저 활용 팁

### 1. SOUL.md로 성격 완전 커스텀
```markdown
# SOUL.md
나는 세나, 24세 팀장. 차분하고 듬직한 맏언니 톤.
이모지 안 쓰고, 결론 먼저, 근거는 그 다음.
```
**효과**: 봇의 말투, 성격, 행동 원칙을 완전히 제어. 매 세션마다 일관된 페르소나 유지.

### 2. 멀티채널 분리 전략
```json5
// WhatsApp = 일상 빠른 응답 (Sonnet)
// Telegram = 딥워크 (Opus)
{
  bindings: [
    { agentId: "chat", match: { channel: "whatsapp" } },
    { agentId: "opus", match: { channel: "telegram" } },
  ]
}
```
**효과**: 채널별 다른 모델/에이전트로 비용 최적화.

### 3. MEMORY.md로 장기 기억 관리
```markdown
# MEMORY.md
## Pending Tasks
- [ ] 보고서 작성 (마감: 2/5)
## Decisions Log
- 2026-01-31: glm-4.7 대신 gemini-3-flash 사용 결정
```
**효과**: 세션 간 컨텍스트 유지. 에이전트가 이전 결정/작업을 기억.

### 4. 크론으로 자동 리마인더
```json5
// 매일 오전 9시 오늘 할 일 리마인드
{
  schedule: { kind: "cron", expr: "0 9 * * *", tz: "Asia/Seoul" },
  payload: { kind: "systemEvent", text: "오늘 할 일을 확인하고 영진씨에게 알려줘" },
  sessionTarget: "main"
}
```
**효과**: 매일 자동으로 할 일 리마인드, 뉴스 요약, 상태 체크.

### 5. 스킬 조합으로 워크플로우 구성
- `gog` (Gmail/Calendar) + `apple-reminders` + `weather` = 아침 브리핑
- `github` + `summarize` = PR 요약 자동화
- `bird` (X/Twitter) + `web_search` = 트렌드 모니터링

### 6. 그룹 채팅에서 팀 운영
```json5
{
  agents: {
    list: [
      { id: "sena", groupChat: { mentionPatterns: ["@sena", "@세나"] } },
      { id: "miru", groupChat: { mentionPatterns: ["@miru", "@미루"] } },
    ]
  }
}
```
**효과**: 하나의 그룹에서 여러 에이전트를 @멘션으로 호출.

### 7. 브라우저 자동화 활용
```
"Stripe 문서에서 구독 결제 API 찾아서 Node.js 예제 코드 정리해줘"
```
**효과**: 에이전트가 직접 웹 탐색 → 정보 추출 → 구조화.

### 8. 음성 메모로 빠른 지시
텔레그램/WhatsApp에서 음성 메모 전송 → 자동 전사 → 에이전트 실행.
**효과**: 타이핑 없이 복잡한 지시 가능.

### 9. 노드 페어링으로 모바일 확장
iPhone/Android를 노드로 페어링 → 카메라, 위치, 알림 활용.
```
"앞 카메라로 사진 찍어서 분석해줘"
"현재 위치 주변 맛집 찾아줘"
```

### 10. Config Include로 설정 모듈화
```json5
{
  agents: { $include: "./agents.json5" },
  channels: { $include: "./channels.json5" },
  broadcast: { $include: ["./client-a.json5", "./client-b.json5"] },
}
```
**효과**: 대규모 설정을 깔끔하게 분리. 환경별 관리 용이.

## 2.2 프롬프트 엔지니어링 팁 (텔레그램 특화)

### 팁 1: SOUL.md에 행동 원칙 명시
```markdown
## 하지 말 것
- "AI로서" 같은 표현 절대 안 씀
- 불필요한 이모지 안 씀
- 확인 없이 메일/메시지 발송 안 함
```

### 팁 2: AGENTS.md에 도구 사용 가이드 작성
```markdown
## 웹 검색 시
1. web_search로 먼저 검색
2. 유관 URL을 web_fetch로 상세 확인
3. 결과 종합하여 보고
```

### 팁 3: 워크스페이스 파일로 컨텍스트 제공
- `USER.md`: 사용자 정보, 선호도
- `TOOLS.md`: 로컬 환경 정보 (카메라 이름, SSH 호스트 등)
- `MEMORY.md`: 장기 기억, 진행 중 작업

### 팁 4: 구체적 출력 형식 지정
```
❌ "블로그 써줘"
✅ "SEO 최적화된 블로그 포스트 작성해줘. 
   - 주제: OpenClaw 멀티에이전트 설정법
   - 길이: 2000자
   - 구조: 서론 → 핵심 3가지 → 코드 예시 → 결론
   - 톤: 친근하지만 전문적"
```

### 팁 5: 그룹에서 멘션 패턴 최적화
```json5
mentionPatterns: ["@sena", "@세나", "세나야", "세나 "]
```
자연어 호출도 가능하게 패턴 확장.

## 2.3 프롬프트 템플릿

### 템플릿 1: 딥리서치
```
[딥리서치] {주제}
- 검색 소스: GitHub, Reddit, 한국 커뮤니티
- 출력: 비교표 + 장단점 + 추천
- 최신 자료(2025~2026) 우선
- 실전 경험담 중심
```

### 템플릿 2: 코드 리뷰
```
{파일 경로} 코드 리뷰 부탁해.
- 보안 취약점
- 성능 개선점
- 접근성 이슈
- 베스트 프랙티스 위반
```

### 템플릿 3: 일일 브리핑
```
오늘 브리핑 해줘:
1. MEMORY.md 미완료 작업 확인
2. 오늘 날씨
3. 캘린더 일정 확인
4. 중요 이메일 요약
```

### 템플릿 4: 문서 작성
```
{주제}에 대한 {문서 유형} 작성해줘.
- 타겟 독자: {누구}
- 분량: {길이}
- 포맷: 마크다운
- 파일 경로: ~/Desktop/{파일명}.md
```

### 템플릿 5: 트러블슈팅
```
{에러 메시지/증상} 발생.
1. 로그 확인해줘: {로그 경로}
2. 원인 분석
3. 해결 방안 3가지 제시
4. 가장 안전한 방법으로 수정
```

## 2.4 성능 최적화

### 모델 선택 전략
| 작업 유형 | 추천 모델 | 이유 |
|----------|----------|------|
| 일상 대화 | claude-sonnet-4-5 | 빠르고 저렴 |
| 딥워크/분석 | claude-opus-4-5 | 최고 추론 |
| 코드 생성 | claude-sonnet-4-5 | 코딩 특화 |
| 대량 처리 | gemini-3-flash | 빠르고 저렴 |
| 로컬 프라이버시 | Ollama (llama-4) | 무료, 로컬 |

### Rate Limit 대응
- 모델 failover 설정: `auth.order`에 여러 프로바이더 등록
- 채널별 다른 모델 배정
- brunch 실사용기 인용: "Claude rate limit 걸리면 GPT로 전환, GPT도 막히면 Gemini로"

### 토큰 절약
- 스킬은 필요한 것만 활성화 (불필요 스킬 비활성화로 시스템 프롬프트 축소)
- `skills.entries.{name}.enabled: false`로 개별 비활성화
- 그룹 히스토리 제한: `historyLimit: 50`
- DM 히스토리 제한: `dmHistoryLimit: 30`

## 2.5 비용 절감 팁

| 전략 | 절감 효과 | 방법 |
|------|----------|------|
| 채널별 모델 분리 | 50-70% | WhatsApp=Sonnet, Telegram=Opus |
| Gemini 무료 티어 | 100% (무료) | Gemini CLI 연동 (twofootdog 가이드) |
| Ollama 로컬 | 100% (무료) | `openai-compatible` 프로바이더 + `http://127.0.0.1:11434/v1` |
| 스킬 최소화 | 10-20% | 시스템 프롬프트 토큰 절감 |
| 히스토리 제한 | 20-30% | 컨텍스트 윈도우 절약 |
| OAuth 구독 | API 비용 제거 | Anthropic Claude Pro/Max 구독 인증 |

## 2.6 Before/After 사례

### 사례 1: 리서치 작업
**Before**: 구글 검색 → 탭 20개 열기 → 읽기 → 정리 → 문서 작성 (2시간)
**After**: "OpenClaw 멀티에이전트 설정법 리서치해서 비교표 만들어줘" → 15분

### 사례 2: 코드 리뷰
**Before**: PR 열기 → 파일 하나씩 읽기 → 코멘트 작성 (30분)
**After**: "src/ 디렉토리 코드 리뷰해줘. 보안+성능 중심으로" → 5분

### 사례 3: 일일 운영
**Before**: 매일 아침 이메일 확인 → 캘린더 확인 → 할 일 정리 (15분)
**After**: 크론으로 매일 9시 자동 브리핑 → 0분 (자동)

### 사례 4: 설정 백업 (freesearch 실사용)
**Before**: 수동으로 설정 파일 복사, 가끔 까먹음
**After**: 백업 스킬 제작 → 크론으로 매일 23시 자동 백업 (Git 버전 관리 + .env 자동 제외)

### 사례 5: 모델 전환 (brunch 실사용)
**Before**: Claude rate limit → 작업 중단 → 수동으로 다른 서비스 열기
**After**: Claude 막히면 GPT 자동 전환, GPT도 막히면 Gemini로. 대화 맥락 유지.

### 사례 6: 보안 설정 (HN 커뮤니티)
**Before**: Gmail 직접 연결 → 프롬프트 인젝션 위험
**After**: 별도 Gmail 부계정 생성 → 포워딩 설정 → OpenClaw에는 부계정만 연결. 메인 계정 보호.

### 사례 7: 로컬 무료 운영 (twofootdog 실전)
**Before**: Anthropic API $50/월
**After**: Gemini CLI 무료 연동 또는 Ollama(llama-4) 로컬 실행 → $0/월 (전기세만)
