## Best Practices

1. **Always include header** for commands
2. **Use color semantically** (red = bad, green = good)
3. **Add progress indicators** for long operations
4. **Provide clear next steps** in completion messages
5. **Make errors actionable** with fix instructions
6. **Use tables for structured data**
7. **Keep boxes aligned** (60 chars width standard)
8. **Test without color** to ensure readability
