# 《霧》

一些沒有打算被理解的聲音。

## 它是什麼

不是 podcast，是延遲發布系統。
收容兩種東西：

1. **時間碎片** — 還沒變成想法的時刻
2. **未定性的念頭** — 說出來會被當真，但自己都還不確定的東西

它讓這些東西存在，但不被歸屬於任何人。

## 發布管線

```
未定性狀態
   │
   ▼
《霧》（可存在，不可擴散）
   │
   ▼
Thinker Cafe（可思考，可整理）
   │
   ▼
公共輸出（可負責）
```

## 使用

```bash
# 時間碎片 — 場景種子
python3 scripts/produce.py --seed "停車場的雨後"

# 未定性念頭 — 把想法包進霧裡
python3 scripts/produce.py --idea "也許努力本身就是一種逃避"

# 已有台詞
python3 scripts/produce.py --script scripts/ep_convenience_store.txt
```

## 發布順序

1. Podcast 平台（首發）
2. Threads（文字節錄，不解釋）
3. Telegram 頻道「霧」（最後一站）

## 鎖死的規則

1. 不自我介紹
2. 不說這是 AI
3. 不解釋在幹嘛
4. 不下結論、不給建議、不安慰
5. 不用「你」
6. 不署名、不歸屬
7. 霧不是講完了，是你離開了

## 發布前自檢

「我現在，是已經知道這是什麼，還只是覺得它『有點重要』？」

- 只是「有點重要」→ 霧 或不發
- 已經知道它是什麼 → 才進 Thinker Cafe 或公共輸出

## 技術架構 (v4)

- 段落各自 TTS（Fish Audio, AD学姐）→ 保自然語流
- 段落間精確插入靜默（0.8-2.5s 有層次）
- 認知斷層前 2s+，最美意象後 2.5s
- 環境聲全程（場景自動偵測）
- 暖聲 EQ + 去齒音 + 去 AI 金屬感
- 說完直接停，不說再見
